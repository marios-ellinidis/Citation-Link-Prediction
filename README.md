# Large-Scale Graph & NLP Citation Prediction Pipeline

## 📌 Overview
This project tackles the complex task of link prediction within a massive academic citation network containing **138,499 nodes and 1,091,955 edges**. The objective is to predict whether one research paper cites another by combining localized topological graph metrics with deep semantic natural language processing.

The final solution is a highly optimized, hybrid machine learning pipeline that fuses **Graph Neural Networks (GNNs)**, **Node2Vec**, and **Transformer-based NLP embeddings** (Specter2, SciBERT) into an extensively fine-tuned ensemble classifier. 

## 🛠 Advanced Data Engineering & Preprocessing

### 1. Strict Graph Data Leakage Prevention
In link prediction, message-passing algorithms and random walks can easily "cheat" if validation edges are present in the training graph. To guarantee rigorous evaluation:
* The dataset was strictly split into training and validation sets **prior** to constructing the `NetworkX` directed graph. 
* All deep structural embeddings (`Node2Vec` and `GraphSAGE`) were strictly derived from the training topology to prevent test-set information leakage.
* A balanced dataset was generated by creating negative edges (non-citations) equal to the number of positive edges, with explicit exclusion checks against the Kaggle test set.

### 2. Multi-Level NLP Embedding Imputation
A significant challenge was handling papers with missing or empty abstracts. To prevent data loss and avoid padding with uninformative zero-vectors, I engineered a custom **Multi-Level Fallback Strategy** to impute missing text embeddings using graph neighborhood context:
1. **Abstract:** Direct embedding extraction if the text exists.
2. **1-Hop Neighbor Imputation:** Average embeddings of immediate neighbors in the citation graph.
3. **2-Hop Neighbor Imputation:** Average embeddings of the "neighbors of neighbors".
4. **Author-based Imputation:** Average embeddings of other papers written by common authors.
5. **Global Mean:** The average of all valid dataset embeddings as a final safety net.

### 3. Memory Management & GPU Batching
Extracting high-dimensional embeddings for 130k+ abstracts requires significant computational resources.
* **Transformer Embeddings:** Tokenization and extraction (Specter2, SciBERT) were accelerated using PyTorch on CUDA. To prevent Out-Of-Memory (OOM) crashes during the 768-dimensional extraction, I implemented a buffered chunking pipeline, safely saving partial `.pkl` matrices to disk every 30,000 iterations.
* **PCA Dimensionality Reduction:** To optimize the classifier's RAM usage and remove multicollinearity, deep embeddings were reduced via `sklearn.decomposition.PCA` (e.g., Specter2 reduced from 768 to 250 dimensions).

## 🧬 Feature Engineering Architecture

The feature space was constructed iteratively, scaling inputs via `StandardScaler`, and combining different measurement strategies:

* **Topological Graph Metrics:** Adamic-Adar Index, Preferential Attachment, Clustering Coefficient, Closeness Centrality, and Degree Centrality.
* **Semantic Similarity (Cosine Distances):** Textual relationships were measured by computing the Cosine Similarity between document embeddings using **Specter2 [CLS] tokens**, **SciBERT**, **Doc2Vec**, **Word2Vec**, and **TF-IDF**.
* **Deep Structural Concatenation:** Unlike text features, the output embeddings from **GraphSAGE** and **Node2Vec** were flattened and natively concatenated (e.g., `[...features, *emb_u, *emb_v]`). This allowed the downstream classifiers to learn complex, non-linear structural relationships between node pairs rather than relying on a single distance metric.

## 🤖 Modeling & Exhaustive Fine-Tuning

The evaluation metric for this project is **Log Loss** (Negative Log-Likelihood):

$$L = -\frac{1}{N} \sum_{i=1}^{N} (y_i \log(p_i) + (1 - y_i) \log(1 - p_i))$$

To find the optimal classifier, I conducted exhaustive hyperparameter tuning (GridSearch/ParameterGrid) across 7 different machine learning architectures. Furthermore, foundational representation models (Node2Vec, Doc2Vec, and GNNs) were independently fine-tuned (walk lengths, window sizes, message-passing layers) to yield the highest quality embeddings before classification.

**Evaluated Classifiers:**
1. `LogisticRegression` (Baseline)
2. `RandomForestClassifier` (Tuned depth, estimators, and leaf samples)
3. `XGBClassifier` (Tuned learning rate, subsampling, and colsample_bytree)
4. `LGBMClassifier` (Tuned num_leaves and learning rates)
5. `CatBoostClassifier` (Tuned depth, L2 regularization, and feature sampling)
6. `MLPClassifier` (Tuned hidden layers (e.g., `(256, 128, 64)`), solvers, momentum, and L2 penalties)
7. **`HistGradientBoostingClassifier` (Champion Model)**

**Final Champion Model Architecture (HistGradientBoosting):**
* `learning_rate`: 0.06
* `max_iter`: 140
* `max_leaf_nodes`: 11
* `min_samples_leaf`: 4
* `l2_regularization`: 0.0
* `early_stopping`: True
* **Performance:** Achieved a highly competitive validation Negative Log-Likelihood (NLL) score of **0.1003**.

## 💻 Repository Structure & Compute Allocation
* `link_prediction.ipynb`: The main Jupyter Notebook containing the end-to-end pipeline, including EDA, multi-fallback embedding generation, iterative feature ablation testing, and cross-validated model training.

**Compute Allocation Strategy:**
* **Local Compute:** Graph topological feature extraction, negative sampling, baseline text embeddings (TF-IDF/Word2Vec), scaling, and tree-based model training.
* **Cloud Compute:** The GraphSAGE training, Node2Vec random walks, and Transformer embedding generations were offloaded to cloud compute environments to leverage GPU acceleration and handle memory-intensive operations.
